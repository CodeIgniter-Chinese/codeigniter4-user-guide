############################
向后兼容性说明
############################

我们尽力在开发产品时最大程度地保持向后兼容（BC）。

只有主要版本（如 4.0、5.0 等）才允许破坏向后兼容性。
次要版本（如 4.2、4.3 等）可以引入新功能，但必须在不破坏现有 API 的前提下进行。

不过，代码库尚未完全成熟，某些 Bug 修复可能会破坏兼容性。所有破坏性变更都会在 :doc:`../changelogs/index` 中说明。

*****************************
哪些不属于破坏性变更
*****************************

- 已弃用的项目不在向后兼容（BC）承诺范围内。它们可能会在下一个 **次要** 版本或之后被移除。例如，如果某个项目自 4.3.x 起被弃用，则可能会在 4.5.0 中移除。
- 在 **system/Language/en/** 中定义的系统消息不受向后兼容性承诺约束。因为这些消息的内容并不属于公共 API。测试应针对函数调用（``lang('...')``），而不是具体文本内容。
- `命名参数 <https://www.php.net/manual/zh/functions.arguments.php#functions.named-arguments>`_ 不被视为 API 的一部分。因此，在必要时我们可能会更改参数名称，以改进代码库。
