## 10. 附录：审查方法说明

```markdown
## 10. 附录：审查方法说明

### 信息点对账方法

审查阶段采用以下四步法进行信息点对账：

1. **信息点提取**：
   - 从英文原文提取所有信息点（事实/条件/限制/步骤/例外/警告/原因/结果）
   - 特别标注限定词（must/should/only/except/required/not allowed）
   - 标注技术细节（数值/版本/参数/返回值/类型）

2. **中文重组核对**：
   - 检查每个信息点在中文中是否有对应表达
   - 标注遗漏/增添/篡改的信息点

3. **语气强度匹配**：
   - 检查限定词语气是否一致
   - must → 必须/一定要（强语气）
   - should → 应该/建议（中语气）
   - may/can → 可以/可能（弱语气）

4. **细节哨兵**：
   - 检查所有数值/版本/参数是否完整保留
   - 检查技术术语是否保持英文

### 可读性审查方法

审查阶段采用以下标准评估中文可读性：

- **清晰性**：句子是否简洁直接，避免名词化和从句堆叠
- **主动性**：优先主动语态，被动语态仅在语义需要时使用
- **逻辑性**：信息顺序符合中文习惯（先结论，后细节）
- **一致性**：并列结构和列表句式统一

### 格式审查方法

审查阶段使用以下脚本进行格式检查：

- 标题装饰线长度检查（≥ 标题长度）
- 代码块缩进一致性检查（应为 4 空格）
- reST 指令完整性检查（不应翻译指令名）
- 交叉引用完整性检查（不应翻译角色名）
- 排版规范检查（间距/标点/大小写）
```
